### Testing Philosophy

Tests must be **antagonistic** - they try to break the code, not just verify happy paths.

Every module/unit needs:
- 10-15 tests minimum
- Null/undefined/empty input tests
- Concurrent scenario tests
- Error condition tests
- Edge case tests

## Examples

All examples must execute without errors and produce correct output (no `undefined`, `NaN`, or wrong values).
